# Инструкция: Панель управления PM-RU Content Agent

## Что это?

Веб-панель для запуска автоматической сборки карточек для маркетплейсов.  
Ника заполняет форму ТЗ → нажимает кнопку → система сама собирает все слайды.

---

## Как запустить?

### Первый запуск

1. Открой папку проекта в Проводнике:  
   `C:\Users\user\Dropbox\Public\Cursor\PM-RU_ContentAgent`

2. Дважды кликни на файл `start_admin.py`  
   (или правой кнопкой → «Открыть с помощью Python»)

3. В браузере автоматически откроется:  
   **http://localhost:8001/admin**

---

## Как создать задание?

### Шаг 1 — Нажми «Новое задание»
Большая синяя кнопка в правом верхнем углу.

### Шаг 2 — Заполни форму ТЗ

| Поле | Что вписывать | Пример |
|---|---|---|
| **Категория** | Стол или Стул | Стол |
| **Артикул** | Артикул из Redmine | 184109 |
| **Вариант/цвет** | Цвет или модификация | Белый глянец |
| **Бренд** | Название бренда | БЕННИ |
| **Название товара** | Полное название | Стол обеденный раскладной |
| **Ш × Г × В (см)** | Три размера | 120 × 75 × 75 |
| **Высота сиденья** | Только для стульев | 46 |
| **Столешница** | Материал столешницы | ЛДСП 22 мм |
| **Цвет/отделка** | Цвет поверхности | Дуб Сонома |
| **Ножки** | Материал ножек | Металл, порошковая окраска |
| **УТП 1–5** | Преимущества товара (3–5) | Антисцарапин |
| **Фото 1–5** | URL фотографии | https://disk.yandex.ru/... |

> 📌 Поля со звёздочкой обязательны. Остальные — по наличию.

### Шаг 3 — Нажми «Передать в пайплайн»

---

## Что происходит дальше?

Карточка задания появляется на дашборде. Видно 9 этапов:

```
Манифест → Валидация → Нормализация → Сжатие →
Шаблон → Рендер → Экспорт → Загрузка → Готово
```

Цветные точки показывают прогресс в реальном времени:
- ⚪ Серая = ожидает
- 🔵 Синяя = выполняется сейчас
- 🟢 Зелёная = выполнен
- 🔴 Красная = ошибка

---

## Где забрать готовые слайды?

Когда задание переходит в статус **«Готово»** — под карточкой появляются кнопки:

- **Слайд 1, Слайд 2...** — открывают/скачивают PNG напрямую
- **Папка с результатами** — копирует путь к папке

---

## Статусы заданий

| Статус | Значение |
|---|---|
| ⚪ Ожидает | Задание создано, пайплайн запускается |
| 🔵 В работе | Идёт обработка, виден текущий шаг |
| 🟢 Готово | Все слайды собраны, можно скачивать |
| 🔴 Ошибка | Что-то пошло не так, текст ошибки отображается |

## Частые вопросы

**Сколько времени занимает одно задание?**  
~20–60 секунд для 4–8 слайдов (зависит от скорости Figma API).

**Можно ли создавать несколько заданий одновременно?**  
Да, все они отображаются на дашборде и обрабатываются параллельно.

**Задание показывает «Ошибка» — что делать?**  
Навести на карточку и прочитать текст ошибки. Чаще всего — некорректный URL фото или временная недоступность Figma. Нужно создать задание заново.

**Где хранятся готовые PNG?**  
Временно в папке `%TEMP%` на сервере. Постоянное хранилище (Yandex S3 / Disk) подключается на следующем этапе.
